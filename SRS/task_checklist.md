# SegmentByMotion 任务清单（SRS）

## 1. 目标
- 建立“GUI 下发任务 -> 节点处理 -> 回传结果 -> 服务端验收落盘”的完整闭环。
- 保障指令通道和数据通道分离，确保控制指令实时性与传输稳定性。

## 2. 范围
- 服务端：任务编排、状态机、双端口 socket、断点续传、结果入库/落盘。
- 节点侧：上线同步、任务确认、资料接收、处理执行、结果回传。
- GUI：仅在任务完成 review 后允许下发，展示节点状态和任务进度。

## 3. 任务清单

### 3.1 协议与通道
- [ ] 完成控制通道消息协议定义（HELLO/TASK_ASSIGN/TASK_CONFIRM/TASK_STATUS_*）。
- [ ] 完成数据通道消息协议定义（CHUNK/CHUNK_ACK/TRANSFER_*）。
- [ ] 明确 `request_id` 幂等规则与重放处理策略。
- [ ] 完成双端口配置项（默认控制端口、数据端口、扩展端口段）。

### 3.2 服务端能力
- [ ] 实现控制通道 socket 服务（节点注册、心跳、任务下发、状态查询）。
- [ ] 实现数据通道 socket 服务（分片发送、ACK、断点续传、校验）。
- [ ] 实现任务状态机流转与持久化（created -> assigned -> ... -> done/failed）。
- [ ] 实现节点注册表与在线状态缓存（含能力信息、当前任务）。
- [ ] 实现结果回传验收与落盘（result.json、heat_export.json/csv、日志）。

### 3.3 GUI 约束与可视化
- [ ] 限制“发送到第三阶段”仅对 `review_done` 任务开放。
- [ ] 新增节点状态面板（在线数、空闲/忙碌、最近心跳）。
- [ ] 新增任务分发进度展示（已下发、传输中、处理中、回传中、完成/失败）。

### 3.4 存储与审计
- [ ] 增加任务分发相关表（节点、分发记录、传输会话、状态事件）。
- [ ] 增加关键状态迁移审计日志。
- [ ] 增加失败原因与重试记录字段。

### 3.5 测试与验收
- [ ] 单元测试：协议解析、状态机、分片校验、断点续传逻辑。
- [ ] 集成测试：节点断线重连、续传恢复、重复消息幂等。
- [ ] 端到端测试：GUI 下发 -> 节点处理 -> 回传落盘 -> 状态完成。
- [ ] 压测：多节点并发任务下发与回传稳定性。

## 4. 里程碑
- M1（协议固化）：完成双通道协议与端口配置，评审通过。
- M2（基础联通）：控制通道可下发任务，数据通道可传输文件。
- M3（可恢复）：断点续传与失败恢复策略可验证。
- M4（闭环上线）：GUI 到结果落盘全链路可用。

## 5. 验收标准
- 可同时接入至少 3 个节点并稳定处理任务。
- 节点异常断开后 30 秒内可恢复到可继续传输状态。
- 单任务完成后回传产物完整落盘且校验通过。
- GUI 可实时查看任务状态与节点状态，不出现状态倒退。
